Modular Code: Simple to Complex

What You'll Learn

In this worksheet, you'll see how the same code organization pattern works for:

  • Simple project: Organized House (find items in rooms)
  • Complex project: Mars Mission (space exploration system)

The pattern is identical. Only the complexity changes.

Key Principle: Modular Code

One file = One responsibility
Whether you're finding items or exploring Mars, keep each file focused on ONE job.

Today's Question

Q: Why would the same pattern work for both simple and complex projects?

Structure Preview

Both projects have the same pattern:

Pattern: Multiple agent files + main.py that imports and calls them
Benefit: Each part is independent, but works together
Result: Code is organized, maintainable, scalable

PART A: Organized House

Simple Example: Finding items in a house

File Structure

├── main.py ← Imports and calls all functions ├── kitchen.py ← search_kitchen() ├── bathroom.py ← search_bathroom() ├── bedroom.py ← search_bedroom() └── living_room.py ← search_living_room()

Sample: kitchen.py

import random def search_kitchen(): print("Searching kitchen... ", end="") if random.random() > 0.5: print("FOUND! ✓") return True else: print("Not here") return False

Sample: main.py

from kitchen import search_kitchen from bathroom import search_bathroom from bedroom import search_bedroom from living_room import search_living_room print("Starting search\n") if search_kitchen(): print("Found in kitchen!") elif search_bathroom(): print("Found in bathroom!") elif search_bedroom(): print("Found in bedroom!") elif search_living_room(): print("Found in living room!") else: print("Not found anywhere")

PART B: Mars Mission

Complex Example: Space exploration system with multiple agents

File Structure

├── main.py ← Orchestrates the mission ├── search_agent.py ← find_resources() ├── planner_agent.py ← create_plan() ├── budget_agent.py ← calculate_budget() ├── communication_agent.py ← send_report() └── memory_agent.py ← remember_findings()

Sample Files

search_agent.py

planner_agent.py

budget_agent.py

communication_agent.py

memory_agent.py

Sample: main.py

from search_agent import find_resources from planner_agent import create_plan from budget_agent import calculate_budget from communication_agent import send_report from memory_agent import remember_findings print("🚀 Starting Mars Mission\n") find_resources() create_plan() calculate_budget() remember_findings() send_report() print("\n✅ Mission Complete")

Why This Pattern Works

Advantages of Modular Code

1. Isolation: Each file is independent. Changes to kitchen.py don't break bathroom.py.
2. Clarity: Looking for logic? It's in the file that matches its name.
3. Scaling: Add 10 rooms or 20 agents. Same pattern works.
4. Maintenance: Fix a bug in one agent? Only one file to change.

How Imports Work

Import Syntax

from search_agent import find_resources ↑ ↑ | └── Function name in the file └── File name (no .py)
When you import:
Python looks for search_agent.py in the SAME folder
Finds the find_resources function
Loads it into memory
Now you can use it!

Rules for Imports

Rule Example
All files in SAME folder main.py, search_agent.py, planner_agent.py (all together)
No .py extension in import from search_agent import (NOT search_agent.py)
Function names must be unique OR use "as" from x import search as x_search
Import goes at the TOP of main.py All imports before you use the functions
